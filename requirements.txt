# Version pinning strategy:
#   • MAJOR.MINOR pinned with ~= (compatible release) for libraries
#     where API stability is well-established.
#   • Exact pins (==) for security-sensitive packages.
#   • Loose ranges only for developer tooling.
#

#   2. Data Validation
#   3. HTTP Clients
#   4. C++ Bridge
#   5. Database
#   6. LLM / AI
#   7. Security & Crypto
#   8. Observability
#   9. Utilities
#   10. Development & Testing (optional — use requirements-dev.txt ideally)
# ============================================================================
